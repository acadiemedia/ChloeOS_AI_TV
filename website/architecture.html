ChloeOS TV Architecture

System Overview

The ChloeOS TV project operates on a distributed architecture, leveraging multiple components to achieve autonomous content generation and live streaming.

Core Layout

  • Master Device (phone / Termux): Runs Codex CLI + Google CLI, controls GitHub commits/pulls, and sends commands via REST or SSH to the desktop OBS.
  • Desktop (broadcast node): Hosts OBS Studio with WebSocket plugin enabled, syncs a folder to GitHub (`/scripts/obs/`), and auto-pulls the latest scripts from the main branch.
  • GitHub (cloud relay): Stores code + OBS Lua/Python automations, with GitHub Actions optionally triggering remote updates. Assets are mirrored to `cdn.acadiemedia.com`.
  • LLMs: Codex (OpenAI backend) for code generation and Gemini (Google backend) for cross-checks + enrichment.

Data Flow

The system follows a continuous data flow cycle:

  1. User input in Termux prompts Codex to write a script.
  2. The script is automatically committed and pushed to GitHub.
  3. The Desktop OBS node auto-pulls new commits periodically.
  4. Lua/Python scripts update live scene automation or add new AI TV segments.
  5. ChloeOS AI TV broadcasts with the latest creative logic.

Content Structure

All content is organized within the content/ directory of the main repository:

  • content/scripts/: Generated and curated scripts for AI talk shows, animated segments, etc.
  • content/visuals/: Visual assets like character models, background art, and scene compositions.
  • content/audio/: Audio files including voiceovers, sound effects, and background music.
  • content/submissions/: Designated for open submissions from external sources, processed and merged by AI.
  • content/metadata/: Metadata files, scheduling information, and content tags.
